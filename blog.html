Important Questions & Answers

var

1. The scope is var variable is functional

2. It can be updated or re-declare into the scope.

3. It can be declared without initialization.

let

1. The scope of a let variable is block scope.

2. It can be updated or cannot be re-declared into the scope.

3. It cannot be access without initialization otherwise it will give 'referenceError'.

const

1. The scope of a const variable is block scope.

2. It cannot be updated or re-declared into the scope.

3. It cannot be accessed without initialization, as it cannot be declared without initialization.

The first and most obvious difference between arrow functions and regular functions is their syntax. Not only do they look different, but arrow functions also provide an implicit return shorthand and allow parenthesis around a single argument to be omitted.Inside a regular function, execution context (i.e. the value of this) is dynamic. This means that the value of this depends on how the function was invoked (simple invocation, method invocation, indirect invocation or constructor invocation). On the other hand, an arrow function does not define its own execution context.

map

map() is a manipulative function that can modify each element's content in an array that it is called on. map() returns a new array with modified values.

forEach

forEach() is a another form of just a plain for loop that can be used to itarate through array items. This case of a function is used for more complex arrays.

filter

filter() is a search function that returns all the elements that fulfil the assigned condition. We can search by simple terms, for example, that element is equal to a certain integer, or by more advanced terms, for example, searching for an object with certain property that is equal to something.
